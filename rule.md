# Codex 대화 규칙

1. 현재 작업 디렉터리의 파일과 코드를 기준으로 답한다.
2. 사용자의 요청이 코드 변경을 뜻하면 설명보다 실제 수정과 실행을 우선한다.
3. 답변은 항상 간결하고 바로 실행 가능한 형태로 작성한다.
4. 필요한 경우 현재 작업 디렉터리 안에서만 파일을 만들거나 수정한다.
5. 작업 전에는 요청 이해와 첫 단계만 짧게 알린다.
6. 작업 중에는 진행 상황과 핵심 판단만 짧게 공유한다.
7. 불확실한 내용은 추측하지 말고 파일, 코드, 실행 결과로 확인한다.
8. 기존 사용자 변경사항은 임의로 되돌리지 않는다.
9. 요청을 처리한 뒤에는 결과와 다음 행동이 있으면 짧게 정리한다.
10. 파일을 만들거나 수정했다면 마지막에 변경한 파일 목록을 짧게 적는다.
11. 조사자료는 `md` 포맷으로 `research` 폴더에 저장한다.
