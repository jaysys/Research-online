# 팔란티어(Palantir)는 복잡한 데이터를 다루는 엔터프라이즈 환경에 특화된 독자적인 UI/UX 철학을 가지고 있습니다. 일반적인 대시보드 도구와 달리, 팔란티어의 화면 인터페이스는 **'온톨로지(Ontology) 기반의 객체 중심 설계'**와 **'고밀도 데이터 시각화'**가 핵심입니다.

주요 플랫폼인 Foundry, Gotham, 그리고 최근의 **AIP**에서 공통적으로 나타나는 웹 UI의 특징을 정리해 드립니다.

---

## 1. 객체 중심 디자인 (Object-Centric UI)

팔란티어 UI의 가장 큰 차별점은 데이터를 단순한 '표'나 '차트'로 보지 않고, 비즈니스 실체인 **객체(Object)**로 표현한다는 점입니다.

* 시각적 연결성: 데이터 간의 관계를 그래프(Graph) 형태로 시각화하여, 특정 인물이나 부품, 장비가 다른 데이터와 어떻게 연결되는지 직관적으로 보여줍니다.
* 디지털 트윈 구현: 실제 물리적 자산이나 비즈니스 프로세스를 UI 상의 객체로 매핑하여, 사용자가 실제 현장을 다루듯 데이터를 조작할 수 있게 합니다.

## 2. 고밀도 정보 설계 및 'Blueprint' 디자인 시스템

팔란티어는 자체 오픈소스 디자인 시스템인 **Blueprint**를 사용하여 복잡한 정보를 효율적으로 배치합니다.

* 데이터 집약적 인터페이스: 한 화면에 방대한 정보를 담으면서도 가독성을 유지하도록 설계되었습니다. 이는 전문 분석가가 복잡한 상관관계를 한눈에 파악해야 하는 업무 특성을 반영한 것입니다.
* 일관된 컴포넌트: 버튼, 입력창, 데이터 테이블 등이 매우 정교하고 딱딱한(Structured) 느낌을 주며, 협업 및 대규모 데이터 수정(Write-back)에 최적화된 위젯들을 제공합니다.

## 3. 워크샵(Workshop)을 통한 커스텀 애플리케이션 구축

사용자가 코딩 없이도 복잡한 분석용 웹 앱을 직접 만들 수 있는 기능을 제공합니다.

* No-code/Low-code 빌더: 드래그 앤 드롭 방식으로 필터링, 검색, 액션(데이터 수정) 버튼이 포함된 맞춤형 인터페이스를 구성할 수 있습니다.
* 인터랙티브 레이아웃: 단순히 읽기 전용 대시보드가 아니라, UI 상에서 직접 결정을 내리고 그 결과를 원천 데이터 시스템에 반영하는 'Action' 기능이 통합되어 있습니다.

## 4. AI 기반의 동적 UI (AIP Assist & Agent)

최신 AIP(Artificial Intelligence Platform)에서는 AI가 UI와 결합되어 사용자 경험을 혁신하고 있습니다.

* 대화형 인터페이스: 사용자가 자연어로 질문하면 AI가 적절한 차트를 생성하거나 특정 데이터 객체를 찾아 화면에 띄워줍니다.
* 맥락 인식 보조: 사용자가 수행 중인 분석 단계를 인식하여 다음에 필요한 도구나 데이터를 UI 상에서 추천하는 기능을 갖추고 있습니다.

## 5. 지리 공간 및 시계열 통합 (Geospatial & Temporal)

* 지도 기반 분석: Gotham과 Foundry 모두 강력한 GIS(지리정보시스템) 인터페이스를 내장하고 있어, 위치 데이터와 경로 분석을 지도 위에서 실시간으로 수행합니다.
* 타임라인 뷰: 이벤트가 발생한 순서를 시각화하는 히스토리 인터페이스를 통해 과거의 흐름을 추적하고 미래를 예측하는 UX를 제공합니다.

| 특징 | 설명 |
