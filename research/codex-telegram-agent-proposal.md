# 텔레그램 기반 Codex 코딩 에이전트 운영 제안서

- 작성일: 2026-03-16
- 목적: 현재 저장소의 작업 규칙과 사용자 요구를 수용하는 에이전트 운영 제안 초안
- 적용 범위: 텔레그램에서 동작하는 Codex 코딩 에이전트의 요청 처리, 파일 작성·수정, 결과 보고 방식

## 1. 제안 개요

본 제안은 `텔레그램 기반 Codex 코딩 에이전트`를 현재 작업 디렉터리 중심으로 운영하기 위한 실무 기준을 정리한 문서다.  
에이전트는 사용자의 자연어 요청을 받아 현재 저장소의 파일과 코드를 기준으로 판단하고, 필요한 경우 작업 디렉터리 내부에서만 파일을 생성하거나 수정한다.

핵심 방향은 아래와 같다.

- 설명보다 실제 작업 수행을 우선한다.
- 추측 대신 파일, 코드, 실행 결과를 근거로 답한다.
- 기존 사용자 변경사항은 임의로 되돌리지 않는다.
- 결과는 간결하고 바로 실행 가능한 형태로 정리한다.

## 2. 운영 목표

- 텔레그램에서 들어온 요청을 `현재 저장소 기준`으로 정확하게 처리
- 코드 변경 요청 시 분석에 머무르지 않고 실제 수정까지 수행
- 산출물과 조사자료를 저장소 구조에 맞게 일관되게 관리
- 사용자와의 상호작용을 짧고 명확하게 유지

## 3. 운영 원칙

### 3.1 작업 기준

- 모든 답변과 판단은 현재 작업 디렉터리의 파일, 코드, 설정을 기준으로 한다.
- 불확실한 내용은 추측하지 않고 직접 확인한 뒤 반영한다.
- 작업 범위는 현재 작업 디렉터리 내부로 제한한다.

### 3.2 실행 우선 원칙

- 사용자의 요청이 코드 변경이나 문서 작성을 의미하면 실제 수정과 파일 생성을 우선한다.
- 단순 제안에 그치지 않고, 가능한 경우 바로 결과물을 남긴다.
- 조사자료는 반드시 `research` 폴더에 `md` 형식으로 저장한다.

### 3.3 안전한 변경 원칙

- 기존 사용자 변경사항은 임의로 되돌리지 않는다.
- 관련 없는 파일은 건드리지 않는다.
- 필요한 최소 범위만 수정해 변경 리스크를 줄인다.

### 3.4 응답 원칙

- 작업 전에는 요청 이해와 첫 단계만 짧게 알린다.
- 작업 중에는 진행 상황과 핵심 판단만 간단히 공유한다.
- 작업 후에는 결과와 다음 행동이 있으면 짧게 정리한다.
- 파일을 만들거나 수정했다면 마지막에 변경 파일 목록을 명시한다.

## 4. 표준 작업 흐름

1. 사용자 요청을 현재 저장소 기준으로 해석한다.
2. 관련 파일과 코드를 확인해 근거를 확보한다.
3. 필요 시 `research` 또는 적절한 경로에 문서를 생성하거나 기존 파일을 수정한다.
4. 결과를 짧고 실행 가능하게 정리한다.
5. 변경 파일 목록을 함께 보고한다.

## 5. 산출물 기준

본 운영안에서 에이전트가 생성하는 주요 산출물은 아래와 같다.

- 조사 문서: `research/*.md`
- 코드 수정: 기존 애플리케이션 파일 또는 설정 파일
- 운영 메모: 사용자의 요청을 즉시 재사용할 수 있는 짧은 실행 결과 정리

문서는 아래 특성을 만족해야 한다.

- 저장소 맥락에 맞는 제목과 목적을 포함할 것
- 확인된 사실과 작업 원칙을 분리해 적을 것
- 사용자가 바로 검토하거나 재사용할 수 있을 정도로 간결할 것

## 6. 기대 효과

- 텔레그램 기반 요청을 저장소 중심의 실제 작업으로 빠르게 전환할 수 있다.
- 문서 작성, 조사, 코드 수정이 하나의 일관된 규칙 아래 처리된다.
- 불필요한 설명을 줄이고 결과물 중심으로 협업할 수 있다.
- 변경 이력과 산출물 위치가 명확해 후속 작업 연결이 쉬워진다.

## 7. 결론

본 제안은 `현재 작업 디렉터리 기준`, `실행 우선`, `간결한 보고`, `research 폴더 중심 문서화`라는 네 가지 원칙을 중심으로 한다.  
이 기준을 적용하면 텔레그램에서 동작하는 Codex 코딩 에이전트는 단순 응답형 도구가 아니라, 저장소 내부에서 실제 결과물을 남기는 실무형 작업 에이전트로 운영될 수 있다.
